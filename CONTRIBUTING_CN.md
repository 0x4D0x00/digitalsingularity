# 贡献指南 - Silicoid Core

感谢您对 Silicoid Core — PotAGI 后端的兴趣！本文档概述了为这个项目贡献的流程。

## 行为准则

本项目遵循行为准则，确保为所有贡献者提供欢迎的环境。在贡献前请阅读完整的[行为准则](CODE_OF_CONDUCT.md)。参与即表示您同意：

- 保持尊重和包容性
- 专注于建设性反馈
- 对错误承担责任
- 对其他贡献者表现出同理心

## 如何贡献

### 1. 报告问题

- 使用 GitHub issue 跟踪器报告 bug 或请求功能
- 提供重现 bug 的详细步骤
- 包含相关代码片段、错误消息和环境详细信息
- 在创建新 issue 前检查现有 issue

### 2. 贡献代码

#### 先决条件
- 后端开发需要 Go 1.20+
- 了解项目架构（见 README.md）
- 熟悉代码库结构

#### 开发设置
1. Fork 本仓库
2. 克隆您的 fork：`git clone https://github.com/YOUR_USERNAME/digitalsingularity.git`
3. 创建功能分支：`git checkout -b feature/your-feature-name`
4. 进行更改
5. 彻底测试您的更改
6. 提交拉取请求

#### 代码风格和标准
- 遵循 Go 约定和最佳实践
- 使用有意义的变量和函数名
- 为复杂逻辑添加注释
- 为新功能编写测试
- 确保代码通过 linting 检查

#### 拉取请求流程
1. 确保您的 PR 包含对更改的清晰描述
2. 引用相关 issue
3. 根据适用性包含测试和文档更新
4. 签署贡献者许可协议（见 CLA.md）
5. 等待审查并处理反馈

### 3. 贡献类型

#### 代码贡献
- Bug 修复
- 新功能
- 性能改进
- 代码重构

#### 非代码贡献
- 文档改进
- 教程创建
- Issue 分流
- 社区支持

## 开发指南

### 项目结构
```
backend/
├── main/           # 主要应用服务
├── silicoid/       # AI 模型集成层
├── aibasicplatform/# 基础 AI 平台服务
├── common/         # 共享工具和安全模块
├── speechsystem/   # 语音处理能力
└── modelcontextprotocol/ # MCP 集成
```

#### 模型集成
- 添加对新 AI 模型提供商的支持
- 改进不同 API 之间的格式转换

#### 平台功能
- 扩展数据库驱动的提示词管理

#### 基础设施
- 添加新的认证方法
- 改进日志和监控
- 增强部署配置

### 测试
- 为新功能编写单元测试
- 修改代码时更新现有测试
- 测试单个组件和集成场景
